0) Objetivo (lo que vas a implementar)

Construir en ReyCAD Engine editor 3D un sistema completo de:

Maniquíes (Templates/Presets)

Humanoide hombre/mujer, arma, perro, dragón, ave, vehículo simple, planeta, árbol, roca, edificio, terreno, mapa/tiles.

Cada maniquí se comporta como “base mesh” con puntos de anclaje (sockets) y opciones de forma.

Skin System (Foto/Color/Decals) Drag & Drop

Arrastras una foto encima del maniquí → genera una skin automáticamente:

Proyección inteligente (planar/tri-planar/cilíndrica).

“Smart wrap” (sin tocar UV a mano).

Ajuste con sliders (escala, rotación, offset, opacidad, blur, roughness, metalness).

Funciona en: armas, personajes, animales, planetas, props, terreno.

Material System tipo Blender (PBR Node-lite)

No haremos un Blender completo, pero sí un Material Editor “Lite pero pro”:

Librería: Metallic, Plastic, Rubber, Glass, Fabric, Skin, Wood, Stone, Lava, Water, Hologram, Mirror, Double-Sided, Toon.

Parámetros: baseColor, roughness, metalness, normal, AO, emissive, opacity, IOR (glass), clearcoat.

Opciones: double face, mirror, transparent, unlit, emissive bloom (lite), subsurface (fake).

OpenFolder / Asset Browser / Library

Abrir carpeta y “montarla” como un proyecto:

Importar glTF/GLB/OBJ/FBX (según disponibilidad), texturas PNG/JPG, HDRI.

Catálogo: models, textures, materials, decals, scenes.

Mantener assets “solo referenciados” (lazy load) para que la app no se muera.

Escenas + Mundo + Terreno + Mapa

Scene graph, layers, prefabs.

Terrain básico: heightmap, brush sculpt, paint splat (4-8 layers), instancing de árboles/rocas.

Diseño ultra ligero

Evitar cargas pesadas: streaming, LOD, instancing, caché, compresión (Draco/meshopt), mipmaps.

2) Arquitectura del sistema “Maniquí + Skin + Material”
2.1 Estructuras base (data models)

Implementa estas entidades:

A) MannequinTemplate

id, name, category (humanoid/weapon/animal/planet/terrain/prop)

baseMeshPath (glb)

sockets[] (nombre + transform)

deformControls[] (sliders para morph targets o escalas)

defaultMaterialsPreset

B) MaterialPreset

id, name, type (pbr/glass/mirror/toon/unlit)

params: baseColor, metalness, roughness, normalStrength, emissive, opacity, ior, doubleSided

textureSlots: albedo, normal, roughness, metalness, ao, emissive, opacity

C) SkinLayer

id

mode: color | photo | decal | pattern

transform: scale/rot/offset

mask: alpha / paintMask

blend: normal / multiply / overlay

target: entireMesh | regionSet | materialSlot

D) ProjectScene

nodes[] (scene graph)

assetsIndex (paths + metadata + hashes)

materials[] (instances)

skins[] (stack)

terrain (opcional)

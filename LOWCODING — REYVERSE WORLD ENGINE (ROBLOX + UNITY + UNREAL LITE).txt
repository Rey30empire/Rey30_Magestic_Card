CODEX: IMPLEMENTAR "Reyverse World Engine"

OBJETIVO:
Crear motor de creación de mundos escalable que combine:

- Motor tipo Roblox (UGC worlds)
- Editor tipo Unity (semi-profesional)
- Render tipo Unreal-lite (cinemático)
- Metaverso competitivo online
- AI transversal cognitiva
- Marketplace de assets
- Física avanzada
- Multiplayer persistente

Arquitectura modular, escalable y versionada.

=========================================================
0) STACK TECNOLÓGICO (USAR LO MÁS POTENTE DISPONIBLE)
=========================================================

FRONTEND / RENDER:

- React + TypeScript
- Three.js (base rendering)
- React Three Fiber
- WebGPU (cuando esté estable)
- @react-three/postprocessing
- three-mesh-bvh (raycast acelerado)
- three-bvh-csg (booleans)
- GLSL custom shaders

FÍSICA:

- Rapier.js (mejor rendimiento que cannon)
- Soporte WASM

MULTIPLAYER:

- Node.js server
- WebSocket (ws)
- Socket.IO (fallback)
- Redis para sync
- Authoritative server model

BACKEND:

- Node.js + Fastify
- PostgreSQL
- Redis
- S3-compatible storage
- Docker

AI CORE:

- Microservicio separado
- Python + FastAPI
- Modelo local o API externa
- Simulación Monte Carlo

CI/CD:

- GitHub Actions
- Docker containers
- Cloud deployment AWS/GCP

0) META

Integrar en el editor 3D de la app un “Mesh Edit Mode” tipo Blender, con herramientas de edición poligonal (quads), más un paquete de herramientas extra (materiales, pintura, UV lite, remesh, retopo simple, openFolder, library, optimización) sin romper el rendimiento.

Regla:

Web/Mobile = modo “Lite” (sin operaciones carísimas, todo con límites).

Desktop = modo “Pro” (más operaciones, workers, openFolder real).

1) STACK Y PRINCIPIOS
Render/Engine

Motor de render: Three.js (o el que ya use el repo, pero mantener API similar).

Geometría editable: BufferGeometry + estructura interna propia (Half-Edge o Winged-Edge).

Workers: todas las operaciones de malla (subdivide, remesh, unwrap, decimate) deben correr en WebWorker (web) o threads (desktop) para no congelar UI.

UI

Viewport central

Panel izquierdo: Asset Browser + OpenFolder

Panel derecho: Tools (Edit Mode, Material, UV, Paint)

Barra superior: modos (Object/Edit/Sculpt/Paint/UV)

Command Palette global (Ctrl+K)

Shortcuts tipo Blender (E extrude, G move, R rotate, S scale, F fill, X delete)

Performance

Operaciones pesadas con límites:

max vertices editable (web): 200k

max faces editable (web): 150k

desktop: 2M si GPU/CPU lo permite

Auto LOD previews (cuando editas, se baja resolución; al soltar, recalcula final)

3) ESTRUCTURA DE DATOS PARA EDICIÓN DE MALLA
3.1 EditableMesh (modelo interno)

Crear módulo EditableMesh que no dependa del render directo.

Representación recomendada: Half-Edge (mejor para operaciones topológicas).

Estructuras:

Vertex { id, position(vec3), normal(vec3), uv(vec2?), edges[], selected }

HalfEdge { id, vertFrom, vertTo, faceLeft, next, prev, twin, selected }

Face { id, halfEdge, materialSlot, selected }

Además:

SelectionState: sets de vertexIds, edgeIds, faceIds

MeshHistory: stack de comandos (undo/redo)

Importante:
Convertir THREE.BufferGeometry -> EditableMesh al entrar en Edit Mode
y EditableMesh -> BufferGeometry al aplicar cambios (o incremental update para performance).

3.2 MeshCommand (Undo/Redo)

Todo cambio de malla debe ser un comando reversible:

AddVerticesCommand

DeleteFacesCommand

ExtrudeCommand

BevelCommand

LoopCutCommand

MergeCommand

etc.

Guardar diffs mínimos:

ids creados

ids borrados

posiciones previas

conectividad previa

4) EDIT MODE: HERRAMIENTAS (MVP DETALLADO)
4.1 Selección

Implementar:

Click: seleccionar elemento bajo cursor (raycast + picking)

Box select (B)

Lasso select (opcional)

Select linked (L): selecciona isla conectada

Select loop (Alt+Click en edge)

Select ring (Shift+Alt+Click)

Picking:

Para faces: raycast a triangles y mapear a faceId

Para edges/verts: usar “pick buffers” (ID render pass) o KD-tree para cercanos.

4.2 Transform (G/R/S)

Gizmo opcional + shortcuts

Pivot: median / individual origins / 3D cursor

Snapping: grid/vertex/edge/face

4.3 Extrude (E)

Extrude funciona para faces/edges:

Para faces:

Duplicar anillo de vertices del perímetro

Crear faces laterales conectando perímetro original con perímetro nuevo

Mover la extrusión por normal promedio o por eje

Para edges:

Duplicar vertices del edge

Crear quad lateral

UI:

“Extrude along normal”

“Extrude distance” slider

“Clamp to axis” toggle

4.4 Fill (F)

Si usuario selecciona:

3–n vertices formando loop → crear face n-gon y triangulación interna

2 edge loops → usar Bridge

4.5 Loop Cut (Ctrl+R)

Implementación MVP:

Detectar edge loop en quads (seguir twin/next en patrón)

Insertar nuevo anillo de vertices a “t” factor (0.5 por defecto)

Dividir faces atravesadas en 2 quads

UI:

scroll cambia factor

click confirma

4.6 Bevel (Ctrl+B)

MVP:

Para edges seleccionados:

Crear offset edges paralelos (approx)

Insertar faces nuevas (quads) entre original y offset

Parámetros:

width

segments (1–4 en web; 1–8 desktop)

4.7 Merge/Weld (M)

Merge by distance:

unificar vertices en radio

actualizar half-edges

Merge to center / to cursor

4.8 Knife (K) Lite

Usuario traza un corte en pantalla

Proyectar la línea sobre la cara intersectada

Split edges/caras creando vertices nuevos en intersección

Limitar: solo una cara o pocas caras para MVP.

4.9 Subdivide / Triangulate / Quadify

Subdivide faces seleccionadas (1–3 iter web)

Triangulate (para export)

Quadify simple (si detecta pares de tris)

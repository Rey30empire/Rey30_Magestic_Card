🧱 DATA STRUCTURE

Implementar Half-Edge usando índices, no punteros.

Estructuras:

pub struct VertexId(pub usize);
pub struct HalfEdgeId(pub usize);
pub struct FaceId(pub usize);
pub struct Vertex {
    pub position: Vec3,
    pub half_edge: Option<HalfEdgeId>,
}
pub struct HalfEdge {
    pub vertex: VertexId,
    pub twin: HalfEdgeId,
    pub next: HalfEdgeId,
    pub prev: HalfEdgeId,
    pub face: FaceId,
}
pub struct Face {
    pub half_edge: HalfEdgeId,
}
pub struct HalfEdgeMesh {
    pub vertices: Vec<Vertex>,
    pub half_edges: Vec<HalfEdge>,
    pub faces: Vec<Face>,
}

No usar referencias mutables circulares.
Todo acceso vía índices.

🚨 INVARIANTES OBLIGATORIOS

Implementar:

pub fn validate_topology(&self) -> Result<(), MeshError>

Debe verificar:

twin.twin == self

next.prev == self

prev.next == self

Cada face tiene ciclo cerrado

Cada face tiene >= 3 edges

No existen half-edges duplicados

No existen vertices huérfanos

No hay non-manifold

No hay caras con área ~0

No hay normales NaN

Si falla → devolver MeshError específico.

📦 1️⃣ REGISTRO DE MALLAS

Tu engine debe mantener un registro interno:

pub struct MeshRegistry {
    meshes: HashMap<String, HalfEdgeMesh>,
}

Cada malla tiene un ID único:

"mesh_001"

"character_body_v1"

etc.

Tu AI nunca accede directo a la malla.
Solo por ID.
